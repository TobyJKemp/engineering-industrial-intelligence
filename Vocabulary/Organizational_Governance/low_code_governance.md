## One-Sentence Summary
Low-Code Governance provides oversight and standards for low-code development—ensuring citizen developers build applications that are secure, maintainable, and aligned with standards.

## The Core Idea
- **Standards:** Standards for low-code applications
- **Review:** Professional review of citizen applications
- **Quality:** Ensure quality and security
- **Governance:** Central governance of platforms
- **Support:** Support citizen developers

## How It Works
1. **Establish standards:** What standards must citizen apps meet?
2. **Select platforms:** Choose approved low-code platforms
3. **Review apps:** Review citizen-developed applications
4. **Provide support:** Support citizen developers
5. **Monitor:** Track applications and monitor for issues

## Think of It Like This
Low-code governance is like **newspaper editors governing citizen journalists**: Editors set standards; review citizen submissions; ensure quality; support citizen journalists.

## The "So What?" Factor
- **Quality:** Ensures citizen apps meet standards
- **Security:** Ensures applications are secure
- **Compliance:** Ensures compliance with policies
- **Support:** Enables citizen development

## Watch Out For
⚠️ **Over-control:** Too much control kills citizen development.
⚠️ **Shadow IT:** Insufficient governance drives shadow systems.
