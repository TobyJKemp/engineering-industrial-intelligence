## One-Sentence Summary
Data Governance establishes policies, processes, and controls for data management—ensuring data quality, security, privacy, and alignment with organizational and regulatory requirements.

## The Core Idea
- **Data policy:** Rules for data management
- **Quality:** Ensuring data quality
- **Security:** Protecting data from unauthorized access
- **Compliance:** Meeting regulatory requirements
- **Ownership:** Clear data ownership and accountability

## How It Works
1. **Define data governance:** Who decides about data?
2. **Establish policies:** Rules for data management
3. **Classify data:** What types of data exist?
4. **Set controls:** Controls on data access, use, retention
5. **Monitor:** Ensure compliance

## Think of It Like This
Data governance is like **library governance**: Policies on book acquisition, cataloging, access, retention; ensuring library serves users properly.

## The "So What?" Factor
- **Compliance:** Meets regulatory requirements
- **Quality:** Ensures data quality
- **Security:** Protects sensitive data
- **Trust:** Data trusted by users

## Watch Out For
⚠️ **Over-governance:** Too restrictive; prevents legitimate use.
⚠️ **Under-governance:** Insufficient control; creates risks.
⚠️ **Siloed governance:** Different data governance for different departments.
⚠️ **Compliance focus only:** Governance only focuses on compliance, misses value.

## Practical Checklist

Before implementing data governance:
- [ ] Have you classified data by sensitivity and value?
- [ ] Are data ownership and stewardship roles assigned?
- [ ] What policies govern data collection, storage, and use?
- [ ] Are data quality standards defined?
- [ ] How do you manage access to sensitive data?
- [ ] Is there compliance monitoring for data regulations?
- [ ] Are data retention and deletion policies in place?
- [ ] How do you handle data lifecycle management?
- [ ] Can you audit data access and use?
- [ ] Is there process for data requests and access?
- [ ] How do you handle data breach scenarios?
- [ ] Can you demonstrate compliance to regulators?

## Quick Decision Guide

**Implement Data Governance when:**
- Organization collects regulated or sensitive data
- Data is critical business asset
- Compliance requirements apply (GDPR, HIPAA, etc.)
- Data quality issues are causing problems

**Phase governance implementation when:**
- Starting with highest-risk data
- Building over time as maturity grows
