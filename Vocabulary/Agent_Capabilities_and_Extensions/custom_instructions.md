# Custom Instructions

## One-Sentence Summary
Custom instructions are user- or developer-supplied directives that shape agent behavior, responses, or capabilities, enabling tailored and adaptive interactions.

## Why This Matters to You
If you want agents to act according to specific needs, preferences, or scenarios, custom instructions are essential. They enable personalization, compliance, and advanced use cases.

## The Core Idea
### What It Is
Custom instructions are:
- Directives or prompts provided at runtime or configuration
- Used to modify agent behavior, tone, or capabilities
- Often dynamic and context-sensitive

### What It Isn't
It is not static configuration or hardcoded logic. True custom instructions are flexible, user-driven, and can be changed without code changes.

## How It Works
1. **Define Instructions**: Specify directives in configuration, UI, or API
2. **Inject at Runtime**: Supply instructions to the agent or model as needed
3. **Adapt Dynamically**: Update or override instructions based on context or user input

## Think of It Like This
Like giving a personal assistant special instructions for a task or meeting.

## The "So What?" Factor
**If you use this:**
- More relevant and effective agent behavior
- Easier adaptation to new requirements
- Better user satisfaction

**If you don't:**
- Agents are rigid and less useful
- Harder to meet diverse needs

## Practical Checklist
- [ ] Are instructions easy to define and update?
- [ ] Are they injected at the right time?
- [ ] Is there a way to override or remove instructions?

## Watch Out For
⚠️ Conflicting or ambiguous instructions
⚠️ Security risks with user-supplied directives
